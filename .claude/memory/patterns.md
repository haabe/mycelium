### Core Pages mapping for content-heavy products (Halland)

For products with page-based content architecture (courses, publications, media sites, service websites), map core pages using Halland's Core Model before delivery. Identify where user tasks and business objectives intersect, then design inward paths (how users arrive) and forward paths (where they go next — context-sensitive to emotional state). Design core pages first, homepage last. Complements Downe P10 (no dead ends) and Allen's User Needs Mapping. Template in `services.yml#core_pages`.
