# Decision Log

**Audience**: internal — published as audit trail, not as public reading.

Record of significant decisions made during product development. Decisions are immutable once logged -- if a decision is reversed, log a new entry that references the original.

## Format

```
### [DATE] - [SHORT TITLE]
- **Diamond**: [ID, scale, phase]
- **Decision**: What was decided.
- **Why_not_alternatives** (structured, per-alternative): list every option considered and a one-line rejection rationale per option. Contrastive explanations ("why X rather than Y") land harder than purely positive ones (Liao et al. 2020); this field must be populated for any decision with ≥1 real alternative. Format:
    - `Option A: <one-line rejection rationale>`
    - `Option B: <one-line rejection rationale>`
- **Theory**: Which framework/theory informed this decision (e.g., "Cynefin - Complex domain", "Cagan - Four Risks").
- **Evidence**: What data or research supports this decision.
- **Confidence**: [0.0-1.0] How confident are we, and what would change this.
- **Scenario ref**: [optional — which scenario from scenarios.yml does this decision serve? Keeps architecture decisions connected to user context. Hoskins: "Draw a line between the problem and the why."]
- **Reversibility**: [easily reversible | costly to reverse | irreversible]
```

### 2026-06-01 — Human-centric mental-model docs layer: SCOPE OUT (it already exists)
- **Diamond**: framework meta-dogfood (docs surface); no product diamond
- **Decision**: Do NOT add a new human-centric "mental models" docs layer separate from the 49 agent-facing skills. **Option (c): explicitly scope out.** The human-centric layer Breunig argues for already exists in `docs/` and is discipline-enforced. - **Prompt**: Drew Breunig, "What Do Humans Need From Docs?" (2026-05-31, dbreunig.com). Thesis: as agents absorb reference material, human docs should build **mental models**, teach the **art of the possible**, explain the **why**, document **design trade-offs**, and **not optimize for completeness**. Skills double as agent-facing reference; human docs must do a different job.
- **Survey finding (what already maps to Breunig's five)**:
    - *Mental models* → README "How it works" (Scales answer *what am I deciding*; Diamonds answer *how* — same four-phase cycle at every scale) + `philosophy.md` fractal double-loop section (with mermaid). Plus `glossary.md` as the vocabulary substrate.
    - *Art of the possible* → README "What it feels like" (Mentor / Guardrail / Checklist modes table).
    - *The why* → `philosophy.md` is the explicit human-facing rationale doc — it opens "The agent operating manual is CLAUDE.md. This page is the human-facing rationale." Covers why-opinionated, why-theory-grounded, why-in-loop-preventive, why-dogfood.
    - *Design decisions / trade-offs* → `philosophy.md` "What Mycelium does not yet do (single-team scope cut)" + README "Who it's not for." Each names the deliberate exclusion and the trigger to revisit.
    - *Don't optimize for completeness* → `docs/README.md` already enforces this: progressive-disclosure layers (Nielsen 30s/5min/30min/deep), length budgets ("over-cap = split, not shrink"), anti-marketing voice rules, information-scent link discipline.
    - The **agent-reference layer is already separate**: `docs/skills/`, `CLAUDE.md`, `.claude/engine/`, `.claude/harness/` are the reference surface; the conceptual docs above are the human surface. Breunig's split is structurally present, not missing.
- **Why_not_alternatives**:
    - `Option (a) add a new human-centric mental-model layer`: rejected — duplicative. `philosophy.md` already IS the why/design-decisions doc and says so; a second "mental models" doc would split the canonical why-surface in two and trip the one-fact-one-place rule.     - `Option (b) extend README / a top-level skill to carry the mental-model job explicitly`: rejected as the headline action — README already carries the mental-model intro (Scales/Diamonds) and art-of-the-possible (feels-like table). Loading more conceptual teaching into README fights its own 250-line hard cap. The job belongs in `docs/`, where it already lives.
    - `Option (c) explicitly scope out and document why`: CHOSEN — the layer exists and is enforced by metadocumentation; the honest move is to record that and stop, not manufacture a new surface.
- **Residual gap named (NOT a new layer — a candidate page within the existing `docs/` layer)**: there is no single doc whose job is the *art-of-the-possible teaching narrative* — "here is how to think with an agent that has gates, and what that lets you do that you couldn't before." Whether to consolidate that into one Phase-2 page (e.g. `docs/mental-model.md` or a section in `get-started.md`) is a separate, smaller decision — deferred, not actioned here. This is a within-layer page question, not a new-layer question.
- **Theory**: Breunig (human docs build mental models, not references); Nielsen (progressive disclosure — already in `docs/README.md`); Pirolli/Card (information scent — already enforced); the framework's own one-fact-one-place cross-link discipline.
- **Evidence**: Direct read of `README.md`, `docs/README.md`, `docs/philosophy.md`, and the `docs/` tree (2026-06-01). N=1 survey by the agent; not validated against a confused human reader. A real newcomer reporting "I read the docs and still don't have the mental model" would be the evidence that flips the residual gap from candidate to actioned.
- **Confidence**: 0.7 — high that the *layer exists* (directly observed); lower that *coverage is sufficient for a cold newcomer* (no outside-reader test of the distributed-mental-model gap). Would increase with a first-time reader's friction log on the conceptual docs.
