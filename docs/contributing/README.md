# Contributing to Mycelium

**Audience**: would-be contributors (issue-openers, PR-authors, friction-finders).
**Time to read**: skeleton (Phase 1); 5 min once filled.
**Last updated**: 2026-05-08.

> This page is forthcoming. Phase 2 will write the contribution process. The voice authority for contributions is [contributing/style.md](style.md).

## What this page will cover

- **The asymmetric ask**: friction-finders get named credit (CONTRIBUTORS.md + receipts cross-link); maintainers retain veto on merging
- **Two paths**:
  1. Open an issue — fastest, lowest commitment, highest signal-per-effort if the issue is well-shaped
  2. Open a PR — for fixes, mechanical improvements, and new artifacts that fit the existing surface
- **What gets credited**: friction → mechanism trace. Surface a real friction; if it shapes the framework, you land on [CONTRIBUTORS.md](../../CONTRIBUTORS.md) and a [docs/receipts/cases/](../receipts/) entry.
- **What gets refused**: feature additions without a real use case behind them. The framework's bias is removal over addition; new mechanisms must show their friction.
- **Voice / style**: [contributing/style.md](style.md).
- **Validation hooks**: `bash .claude/tests/validate-template.sh` + `python3 .claude/scripts/validate_canvas.py`.
