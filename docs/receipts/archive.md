# Receipts archive

**Audience**: practitioners and historians wanting to see older receipts that have rotated off the active list.
**Time to read**: 2 min.
**Last updated**: 2026-05-08.

This page is the long-term archive for case files that are no longer load-bearing on the active receipts list. Currently empty — the framework is too young (oldest active case is 2026-04-tic-tac-toe).

## Archive rules

A case is archived when:

1. **It is more than 12 months old** AND
2. **The pattern it surfaced has not recurred** (no new cluster instance, no related correction in the last 6 months) AND
3. **The mechanism it produced is still load-bearing** (i.e., the case shaped the framework durably; if the mechanism was deprecated, see deprecation rules below)

Archive entries are summary paragraphs (~50 words each) that link to the original case file. The original case files in `docs/receipts/cases/` are NEVER deleted — git history is the canonical record. The archive page is a navigation aid for older receipts.

Manual quarterly review per `/framework-health` step 4c. No automation until N exceeds ~50 cases (premature automation cost).

## Deprecation

If a mechanism a receipt produced was later removed (e.g., a guardrail that turned out to be over-fit), the case file is updated with a "Deprecation" section explaining what changed. The archive entry then notes both the original mechanism and its replacement / removal. Deprecated cases are NOT removed — the lesson the framework learned remains valuable even when the specific mechanism does not.
